أريدك أن تصمم وتبرمج صفحة فواتير احترافية (Invoices Page) داخل نظام إدارة عيادة أسنان، تكون ثنائية اللغة (عربي / English)، RTL عند اختيار العربية، ومتوافقة مع المتطلبات المالية والإدارية في الكويت 🇰🇼، مع التركيز على الجوانب التالية بالتفصيل:

🧭 1️⃣ الهيكل العام للصفحة

صفحة Dashboard مخصصة للفواتير داخل لوحة الإدارة (Admin / Reception).

تحتوي على جدول رئيسي يعرض قائمة الفواتير مع الأعمدة التالية:

🔢 رقم الفاتورة (Invoice No.)

🧍 اسم المريض (Patient Name)

🧑‍⚕️ الطبيب المعالج (Doctor)

📅 التاريخ (Date)

💰 المبلغ الإجمالي (Total Amount - د.ك KWD)

📝 الحالة (مدفوعة / غير مدفوعة / مؤجلة)

🛠️ إجراءات (عرض - طباعة - تعديل - حذف)

أعلى الجدول يوجد شريط أدوات يحتوي على:

زر ➕ لإنشاء فاتورة جديدة

مربع بحث (بحث بالاسم / الرقم / التاريخ)

فلترة حسب الحالة أو الطبيب

اختيار اللغة (🇸🇦 عربي / 🇬🇧 English)

🧾 2️⃣ تفاصيل الفاتورة

عند فتح أو إنشاء فاتورة، يتم عرض نموذج يحتوي على:

🧍 معلومات المريض

الاسم الكامل

الرقم المدني (CID)

رقم الملف الطبي

رقم الهاتف

🧑‍⚕️ معلومات الطبيب

الاسم

التخصص

🧾 تفاصيل الفاتورة

رقم الفاتورة (توليد تلقائي)

تاريخ الإصدار (تلقائي + قابل للتعديل)

المحافظة والمنطقة (Dropdown وفقًا للمحافظات الكويتية: العاصمة، حولي، الفروانية...)

عنوان العيادة (من الإعدادات)

💵 3️⃣ قائمة الخدمات / العلاجات داخل الفاتورة

جدول داخلي يمكن إضافة أسطر فيه (Dynamic Table):

#	الخدمة / العلاج	الكمية	السعر للوحدة (د.ك)	الإجمالي (د.ك)	ملاحظات

زر “➕ إضافة خدمة” لإضافة صف جديد.

السعر بالعملة الكويتية KWD (د.ك)، مع 3 منازل عشرية لدعم الفلس.

حساب المجموع الفرعي (Subtotal) + الخصم (Discount) + المجموع النهائي (Total).

⚠️ لا يتم تطبيق VAT لأن الكويت لا تطبق ضريبة القيمة المضافة حاليًا.
يمكنك إضافة حقل اختياري "رسوم إضافية" (Extra Fees) حسب الحالة.

🌐 4️⃣ الطباعة والتصدير

زر 🖨️ طباعة الفاتورة يفتح نموذج طباعة احترافي يحتوي على:

شعار العيادة + اسمها + العنوان + رقم الهاتف الكويتي (+965)

اسم المريض + رقم الملف + الطبيب

رقم الفاتورة + التاريخ

جدول الخدمات بشكل أنيق مع الحدود

المجموع النهائي بخط كبير وواضح (مثلاً: "الإجمالي: 45.750 د.ك")

ملاحظة في أسفل الصفحة (مثلاً: “الأسعار بالدينار الكويتي - لا تشمل أي ضريبة”)

توقيع المريض + توقيع المحاسب / الطبيب

زر 💾 تصدير PDF يستخدم مكتبة مثل jspdf أو html2pdf.js لإخراج الفاتورة بنفس تصميم الطباعة.

📊 5️⃣ لوحة تحليل الفواتير (اختياري)

قسم جانبي أو صفحة فرعية تعرض إحصائيات شهرية:

إجمالي الإيرادات بالدينار الكويتي (د.ك)

عدد الفواتير

أكثر الخدمات طلبًا

فلترة حسب الطبيب أو المحافظة

⚙️ 6️⃣ المعايير الفنية

الواجهة تستخدم React أو Vue (حسب المشروع الحالي).

التصميم احترافي باستخدام TailwindCSS أو Bootstrap 5 مع دعم RTL.

البيانات تُجلب من الـ API (Laravel أو Node backend).

عند الحفظ يتم إرسال البيانات إلى endpoint مثل:

POST /api/invoices


وتخزينها في جدول invoices و invoice_items.

📝 7️⃣ قواعد السوق الكويتي

العملة الرسمية: KWD – دينار كويتي

لا توجد VAT إلزامية.

التنسيق المالي: 3 منازل عشرية (0.000) للفلس.

التاريخ بالصيغة: يوم/شهر/سنة (d/m/Y).

المحافظات المعتمدة: العاصمة، حولي، الفروانية، الأحمدي، الجهراء، مبارك الكبير.

✅ الهدف النهائي:

إنشاء صفحة فواتير أنيقة، منظمة، قابلة للبحث والطباعة، متوافقة مع النمط الإداري للمجال الطبي في الكويت.

تدعم ثنائية اللغة، RTL، العملة الكويتية، وتكون جاهزة للاستخدام الفعلي في عيادة أسنان.

👌 استخدم أفضل الممارسات في بناء المكونات (Components) وتنسيق الجداول والطباعات.
